## Page 5

Secure Baselines: Secure configuration standards or guidelines that define the

minimum-security settings for systems or devices.

ICS/SCADA (Industrial Control Systems/Supervisory Control and Data Acquisition):

Systems used in industrial automation to control and monitor physical processes.

Embedded Systems: Computers integrated into other devices, like appliances or

machinery.

RTOS (Real-Time Operating System): An operating system designed for real-time and

time-sensitive applications.

IoT (Internet of Things) Devices: Connected devices, such as smart thermostats and

cameras, that communicate over the internet.

Site Surveys: Evaluations of physical locations to determine optimal placement of

wireless equipment.

Heat Maps: Visual representations of wireless signal strength and coverage areas.

Mobile Device Management (MDM): The process of managing mobile devices, apps,

and data within an organization.

Bring Your Own Device (BYOD): A policy that allows employees to use their personal

devices for work purposes.

AAA (Authentication, Authorization, and Accounting): A framework for controlling

access to computer resources.

RADIUS (Remote Authentication Dial-In User Service): A protocol used for centralized

authentication, authorization, and accounting for network access.

Static Code Analysis: The examination of source code for potential security

vulnerabilities without executing the code.

Code Signing: The process of digitally signing software to verify its authenticity and

integrity.

Sandboxing: Isolating and running applications in a controlled environment to prevent

harm to the system.

Enumeration: The process of listing or identifying assets and resources on a network.

Sanitization: The secure deletion or cleaning of data to prevent unauthorized

recovery.

Dynamic Analysis: Evaluating software by running it to observe its behavior.

Package Monitoring: Monitoring software packages and updates for security

vulnerabilities.

Threat Feed: A source of information about current security threats and

vulnerabilities.

## Page 6

Open-Source Intelligence (OSINT): Gathering information from publicly available

sources to assess security risks.

Dark Web: A part of the internet not indexed by search engines, often used for illegal

activities.

Penetration Testing: Ethical hacking to identify vulnerabilities in systems or networks.

False Positive: Incorrectly identifying something as a security threat when it is not.

False Negative: Failing to identify a security threat when it actually exists.

Common Vulnerability Scoring System (CVSS): A framework for assessing the severity

- f vulnerabilities.
Common Vulnerability Enumeration (CVE): A standardized identifier for vulnerabilities.

Exposure Factor: The proportion of a vulnerability's impact that is realized when

exploited.

Risk Tolerance: An organization's willingness to accept risk in its operations.

Segmentation: Isolating and dividing networks or systems to contain security breaches.

Compensating Controls: Alternative security measures used when primary controls are

not feasible.

Audit: A systematic evaluation of security controls, policies, and procedures.

Quarantine: Isolating a potentially compromised system or network to prevent further

damage.

Security Content Automation Protocol (SCAP): A standardized protocol for security

management.

Agents/Agentless: Methods of collecting security data, with or without installed

software agents.

Security Information and Event Management (SIEM): A system for centralizing and

analyzing security event data.

Data Loss Prevention (DLP): Technologies and policies to prevent data leakage.

Simple Network Management Protocol (SNMP) Traps: Notifications sent by network

devices to alert management systems.

NetFlow: A network protocol used for monitoring traffic and collecting data.

## Page 7

IDS/IPS (Intrusion Detection System/Intrusion Prevention System): Security systems

that monitor and protect networks from threats.

Agent-Based: Security solutions that require the installation of specialized software

agents.

Centralized Proxy: A server that acts as an intermediary between users and the internet

for security and content filtering.

Group Policy: A feature in Windows environments for managing user and computer

settings.

SELinux (Security-Enhanced Linux): A Linux security module that enforces access

control policies.

DNS Filtering: Filtering or blocking specific websites or content based on DNS requests.

Domain-Based Message Authentication Reporting and Conformance (DMARC): An

email authentication protocol.

DomainKeys Identified Mail (DKIM): An email authentication method that uses

cryptographic signatures.

Sender Policy Framework (SPF): An email authentication method that helps prevent

email spoofing.

Network Access Control (NAC): A security solution that enforces policy compliance for

devices connecting to a network.

Endpoint Detection and Response (EDR)/Extended Detection and Response (XDR):

Solutions for detecting and responding to security threats on endpoints.

Federation: Integrating identity and access management across different systems and

- rganizations.
Single Sign-On (SSO): A user authentication process that allows access to multiple

systems with a single set of credentials.

Lightweight Directory Access Protocol (LDAP): A protocol for accessing and managing

directory information.

Open Authorization (OAuth): A protocol for secure authorization of applications.

Security Assertions Markup Language (SAML): An XML-based standard for exchanging

authentication and authorization data.

Attestation: The process of confirming the accuracy and validity of information.

Role-Based: Access control based on a user's role or job function.

Rule-Based: Access control based on predefined rules and conditions.

## Page 8

Integrations and Application Programming Interfaces (APIs): Mechanisms for

connecting and interacting with different software applications.

Chain of Custody: Documented tracking of evidence from its collection to its

presentation in legal proceedings.

E-Discovery (Electronic Discovery): The process of identifying, preserving, and collecting

electronic information for legal purposes.
